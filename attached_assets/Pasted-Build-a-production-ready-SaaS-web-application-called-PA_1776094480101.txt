Build a production-ready SaaS web application called “PA MedTrack” for Pennsylvania Personal Care Homes (PCH) and Assisted Living Residences (ALR). The target customer is a medication administration trainer who works with many personal care homes and assisted living facilities in Pennsylvania and wants to offer this software to those facilities as a subscription product.

The software must help facilities track:
1. Medication Administration training
2. Certifications
3. Recertifications
4. Annual practicum requirements
5. Annual direct care training hours
6. Uploaded certificates and proof documents
7. Alerts, reminders, and compliance reports
8. Multi-facility and multi-company access

This should be a multi-tenant SaaS app where:
- One software owner account can manage all customer organizations
- Each customer organization can have multiple facilities
- Each facility can have many employees
- Each employee can have many training, certification, recertification, and document records

The app should feel polished, professional, and ready to sell to real facilities.

==================================================
PRODUCT GOAL
==================================================

Create a Pennsylvania-focused compliance tracking system for personal care homes and assisted living facilities to manage medication technician / medication administration training status for staff, trainers, and supervisors.

This is NOT a generic LMS.
It is a compliance operations platform.

The main value:
- Know who is compliant
- Know what is due soon
- Know what is expired
- Upload proof documents
- Print survey-ready reports
- Prevent missed renewals
- Track multiple buildings easily

1. Platform Owner / SaaS Admin
- Manages all subscribing organizations

5. Employee / Staff User

==================================================
PENNSYLVANIA-SPECIFIC COMPLIANCE LOGIC
==================================================

Create a rules engine that supports Pennsylvania PCH and ALR workflows.

System defaults should include these training types:

1. Medication Administration Initial Course
- category: medication
- applies to staff who administer medications
- default renewal interval: 730 days
- document required: yes

2. Medication Administration Annual Practicum
- category: practicum
- applies to staff who administer medications
- repeats yearly
- document required: yes or optional based on settings, but the system should support upload

3. Medication Administration Trainer Certification
- category: trainer
- applies to trainers
- track initial completion
- document required: yes

4. Medication Administration Trainer Recertification
- category: trainer
- applies to trainers
- renewal interval: 1095 days
- document required: yes

5. Diabetes/Insulin Education
- category: specialty
- renewal interval: 365 days
- document required: yes

6. Annual Direct Care Training Hours - PCH
- category: annual_hours
- required_hours: 12
- applies to PCH direct care staff

7. Annual Direct Care Training Hours - ALR
- category: annual_hours
- required_hours: 16
- applies to ALR direct care staff

The app should let org admins edit these defaults if regulations or client policies change.

Compliance logic:
- If an employee administers medications, they must have a valid Medication Administration Initial Course.
- If the due date is within warning window, mark due_soon.
- If due date passed, mark expired.
- If document required but no file uploaded, surface missing document warning.
- Employees who administer meds should also have annual practicum tracking.
- Employees who are trainers should have trainer certification and trainer recertification tracking.
- For annual training hours, auto-calculate required vs completed by year.
- Facility type should drive annual hour requirements.
- Allow manual overrides with audit logging.
- Allow “not applicable” status where appropriate.

==================================================
MULTI-TENANCY AND SECURITY
==================================================

This must be secure and properly scoped.

Requirements:
- Every organization’s data must be isolated from all others
- Users can only access records they are permitted to see
- Facility managers can only access assigned facilities
- Employees can only view their own records
- Platform admin can see all organizations
- Use strong authorization checks on every route and API endpoint
- Add audit logs for all create, update, delete, verify, and status override actions

Use row-level permissions in app logic and, if supported in the stack, at the database level as well.
